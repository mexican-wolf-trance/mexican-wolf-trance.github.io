SOFTWARE HARDENING

BIOS settings

  • Contains the BIOS code AKA CMOS setup program, which is a program used to configure hardware settings the system
  • CMOS: it is a security best practice to ensure that systems can boot only from the local hard disk and not from the network card, USB drive or CD/DVD-ROM to boot an OS into memory and gain access to the system.
  • Ensure CMOS is configured to only allow booting from hard disk with user ID and password (must be separate from windows password)
  • Use CMOS setup program to disable unnecessary ports

USB drives

  • USB ports can be disabled in CMOS startup program

SOFTWARE HARDENING

Removeable storage

  • Any storage medium that can store data and them be removed from the system such as USB flash drive, FireWire, eSATA, external hard drive
  • Removable Storage can transport worm viruses from the outside to inside the organization's network
  • Company supplied removable storage should be encrypted in case the employee loses the device

SOFTWARE HARDENING

OS hardening

  • The process of removing unnecessary features of the operating system, disabling unnecessary service and removing unnecessary accounts to reduce attack surface
  • Uninstall any unnecessary software from the system
  • A system should be reformatted and a fresh install of the OS applied either manually or through an image
  • Disable Print Spooler Service, Workstation service, Server service, messenger service
❮ ❯

NETWORK HARDENING

Network security hardening

  • Update the firmware on all network devices
  • Always ensure the network is using the most secure devices
    • Use switches instead of hubs, servers newer than MS 2005
  • Update the port security by limiting which systems can connect to a port on the switch by listing specific MAC address with the port
  • MAC limiting and Filtering – filtering allows the administrator to restrict which systems can send data to other systems on the network
  • Disable unused interfaces (ports)
  • 802.1x – for both wired and wireless; ensure anyone who connects to the network supplies valid credentials before the network connections is allowed

NETWORK HARDENING

Group Policies

  • Tool for hardening Windows system; allows administrator to enable or disable various features in Windows
  • Policies include:
    • Local policies
    • Windows Firewall and Advanced Security
    • Software Restriction
    • Advanced Audit Policy Configuration
    • Scripts
    • IE Maintenance
    • Control Panel, Desktop, Start Menu and Taskbar
  • Can be set up as a template and/or group policy or Organizational Unit Policy
  • Can also build security policy as text files with script as a security template and deploy to a group of machines

NETWORK HARDENING

Configuration Baseline

  • A standard configuration that has been approved by the company for a specific type of system or device as being secure. Required for all systems in order to meet the desired security requirements of the company.
  • Any changes to a system must follow the change management system process as defined by the company
  • Baselines vary based on location in the network and how used with the company
  • Security baselines for each type of system must be properly documented and the steps recorded and tested prior to distributing to the security administrators
  • Phases - Security Posture, initial Baseline configuration, continuous security monitoring, remediation
